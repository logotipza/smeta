# Smeta — Контекст проекта (на 2026-06-05)

## Стек
- Frontend: React 18 + Vite + Tailwind CSS v4
- Backend: Node.js + Express
- State: React useState (no global store)
- Excel: SheetJS (`xlsx`) на клиенте
- Dev: frontend 5173, backend 3001

## Архитектура

### Модель данных

#### `workTypes[]` — виды работ (колонки таблицы)
```ts
{
  id: string,
  name: string,           // редактируется inline в шапке
  specialistId: string,   // ссылка на rates
  globalRisk: number,     // общий риск вида работы (default: 1)
}
```

#### `rows[]` — задачи (строки таблицы, плоский массив с иерархией)
```ts
{
  id: string,
  name: string,
  parentId: string | null, // null = корень
  isExpanded: boolean,     // свёрнуто/развёрнуто
  estimates: {
    [wtId]: { clean: number, risk: number }
  }
}
```

#### `rates[]` — справочник специалистов (из `/api/rates`)
```ts
{
  id: string,
  role: string,
  rate: number  // ставка в час
}
```

### Ключевые хелперы (вне компонента)
- `getSubtreeIds(rows, rootId)` — все ID в поддереве
- `getLevel(rows, row)` — глубина вложенности
- `getVisibleRows(rows)` — фильтр свёрнутых + построение дерева
- `hasChildren(rows, rowId)` — есть ли дочерние строки
- `colAddr(idx)` — число → Excel-буква (0=A, 1=B...)

### Калькуляция
- `calcTotal(clean, rowRisk, globalRisk)` — `Math.ceil(clean * rowRisk * globalRisk)`
- `calcCellClean(row, wt)` — для родителей: сумма дочерних; для листьев: own clean
- `calcCellTotal(row, wt)` — для родителей: сумма дочерних total; для листьев: calcTotal

## Ограничения / известные проблемы
- DnD для **строк** отключён (сложно с иерархией; можно вернуть позже)
- Ставка (rate) используется только в Excel и итоговой карточке; в таблице не отображается
- Нет валидации циклических ссылок при indent (есть защита от indent в себя)
- Нет undo/redo
- Нет сохранения сметы на бэкенде (только в памяти frontend)
